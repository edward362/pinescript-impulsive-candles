# Style Guide (Pine v6)

- **Version header** at the top with summary, author, license, version.
- **Inputs**: group logically; readable titles; sensible defaults.
- **Vars**: `lower_snake_case`; inputs in `camelCase` or readable words.
- **Repaint safety**: draw on confirmed bar; document any exceptions.
- **Plots**: consistent colors; avoid visual clutter; tiny shapes for signals.
- **Changelog**: bump versions and update `docs/CHANGELOG.md`.
